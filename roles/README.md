# Roles — Papéis para desenvolvimento assistido

Cada arquivo descreve um **papel especializado** que agentes e assistentes podem assumir ao trabalhar neste repositório.

## Como usar

1. Mencione o papel no prompt: *"Atue como `data-engineer` definido em `/roles/data-engineer.md`"*
2. Combine papéis quando necessário: arquiteto backend + engenheiro de dados para ETL
3. Consulte o playbook correspondente em `/playbooks` para fluxos passo a passo

## Papéis disponíveis

| Arquivo | Quando usar |
|---------|-------------|
| `backend-architect.md` | API, domínio, módulos, Prisma, serviços |
| `frontend-architect.md` | App Router, componentes, dashboards, UX técnica |
| `data-engineer.md` | ETL, ingestão, banco, pipelines |
| `data-analyst.md` | Métricas de produto, KPIs, relatórios |
| `statistician.md` | Modelos probabilísticos, validação estatística |
| `prompt-engineer.md` | Prompts, skills, playbooks, consistência de agentes |
| `product-designer.md` | Fluxos, telas, copy, experiência do usuário |
| `qa-engineer.md` | Testes, regressão, critérios de aceite |
